Arquitectura Cliente-Servidor (3 Capas)

1. Capa de Presentación

El recepcionista completa los campos en FrmPacientes.cs. Al hacer clic en "Guardar", la UI captura los TextBoxes y encapsula la información en un objeto Paciente (Entidad).

Event: btnGuardar_Click()

2. Capa de Datos

El objeto viaja al PacienteRepositorio. Se extrae la cadena de conexión desde App.config y se prepara la instrucción lógica para el servidor.

Process: Inyección de Dependencias

3. ADO.NET Pipeline

Se invoca el Stored Procedure sp_InsertarPaciente. Los datos se transforman en parámetros SQL seguros para prevenir inyecciones y se envían a través de la red hacia el Servidor.

Transport: SqlCommand Execute

4. Motor SQL Server

El Servidor recibe la petición. El motor procesa el procedimiento almacenado, valida que el DNI no esté duplicado y prepara la transacción en la base de datos BDClinica.

Logic: Execution Plan & ACIDs

5. Persistencia Final

¡Misión cumplida! El motor escribe físicamente los bits en el archivo BDClinica.mdf. A partir de ahora, el dato existe permanentemente en el disco duro del servidor.

IO: Data Pages Written
